# Part 6：AIゼロ生成実践 〜 プロンプトだけでBotを丸ごと作る 〜

> **このPartのゴール:** Part 3のコードを「見る」のをやめて、AIとの対話だけでゼロからBotを生み出す体験をする。  
> これが本講座の本当のゴールだ。

---

## 1. なぜ「ゼロ生成」が重要なのか

Part 3 にはおみくじBotの完全なコードが載っている。  
それをコピーして動かすだけなら、今日から10分でできる。

しかし、それはあなたを「コードのコピー職人」にするだけだ。

**バイブコーディングエンジニアの本質は、「思考をコードに変換するプロセスを設計できること」だ。**

Part 6では、あえて Part 3 を見ずに、AIとの対話だけで以下のことを達成する:

```
1. 自分なりのお題を決める（おみくじBotより面白いものでも構わない）
2. ChatGPT でアイデアを発散させる
3. Gemini でブラッシュアップ＋プロンプト設計させる
4. Claude でZIPを生成させる
5. 動かす → 詰まる → AIにデバッグさせる
6. Part 3 のコードと比較して「なぜ違うか」を学ぶ
```

そのプロセスを通じて、「コードを書く力」ではなく「AIと作る力」が身につく。

## 2. お題のアイデア集

### 🎯 おみくじBotより面白いかもしれないお題

Part 3 と同じおみくじBotを作ってもいい。しかし、せっかくなら自分のアイデアで作ってみよう。  
以下はバイブコーディングの練習に適した「ちょうど良い難易度」のお題だ。

#### 🟢 Easy（Part 3 と同等）

```
【タイマーBot】
  /pomodoro start → 25分タイマーを開始してDMで通知
  /pomodoro status → 残り時間を表示

【名言Bot】
  /quote → 偉人の名言をランダムで100パターン表示
  /quote [カテゴリ] → テーマ別（勇気・愛・仕事など）

【サイコロBot】
  /roll [面数] [個数] → 自由なダイスを振る
  /roll 20 → D20で大冒険！

【天気占いBot】※ Tier 2 のお題
  Open-Meteo で今日の天気を取得して運勢に反映
```

#### 🟡 Medium（少し難しい）

```
【英語学習Bot】
  /word → 今日の英単語（TOEIC頻出500問から）
  /quiz → 4択クイズ形式で出題
  /score → 自分の正答率を表示

【RPGステータスBot】
  /status → 自分のキャラクターステータスを表示
  /battle @ユーザー → 簡易バトルシミュレーション
  /level up → 一定ポイントでレベルアップ

【天気 × 献立Bot】
  /dinner → 今日の天気・気温に合った夕食メニューを提案
  Open-Meteo + 献立データベース（JSON）を組み合わせる
```

#### 🔴 Hard（本格的）

```
【AIキャラクターBot】
  /chat → Gemini APIを使って個性のあるキャラクターと会話
  キャラクターの口調・性格をプロンプトで設定

【読書記録Bot】
  /book add [タイトル] [著者] → 読んだ本を記録
  /book list → 自分の読書リスト表示
  /book stats → 月別・ジャンル別統計

【ポイントBot（サーバー経済）】
  メッセージ送信でポイント付与
  /point give @ユーザー → ポイント送金
  /rank → サーバー内ランキング
  → JSONまたはCloud Storageで永続化
```

## 3. ゼロ生成の実践手順（ウォークスルー）

ここでは「英語学習Bot」を例に、実際の手順をウォークスルーする。  
自分のお題に読み替えながら試してほしい。

---

### 🚀 Step 1: ChatGPT でアイデアを発散させる

**ブラウザで ChatGPT を開いて以下を投げる:**

```
【ChatGPTへのプロンプト - Step 1】

Discord Botで英語学習アプリを作りたい。
技術スタック: Node.js + discord.js v14

今思いついている機能:
- /word で今日の英単語を表示
- /quiz で4択クイズ
- /score で正答率

これに追加するとさらに面白くなるアイデアを10個出してほしい。
技術的な実現可能性は気にしなくていい。
ユーザーが「続けたくなる」仕掛けを重視して。
```

**ChatGPTが返してくるアイデアを全部メモしておく。**

次に「絞り込み」を依頼する:

```
【ChatGPTへのプロンプト - Step 1b（絞り込み）】

上記のアイデアの中で、
・初心者が一人で実装できる範囲
・discord.js v14 で実現可能
・ユーザーが毎日使い続けたくなる

この3条件を満たすものをTop 5に絞って、
それぞれ実装の難易度（Easy/Medium/Hard）も教えて。
```

---

### 🔍 Step 2: Gemini でブラッシュアップ

**ブラウザで Gemini を開いて以下を投げる:**

```
【Geminiへのプロンプト - Step 2】

Discord Botで英語学習アプリを作る。
ChatGPTとのやり取りで以下の機能を決めた:

[ChatGPTのTop 5をそのままペースト]

この設計に対して以下の観点でレビューしてほしい:

1. discord.js v14 での実装で技術的に問題はあるか
2. データ設計（どのデータをどう保存するか）の提案
3. UX（ユーザー体験）の観点で改善できる点
4. セキュリティ・プライバシーで気をつけること
5. GCP 無料枠（e2-micro + PM2）での運用で問題が起きそうな点

辛口で評価してほしい。
```

**Geminiのフィードバックを踏まえて要件を修正する。**

---

### 📝 Step 3: Gemini に「Claudeへのプロンプト」を生成させる

これが本講座で最も重要なスキルだ。

```
【Geminiへのプロンプト - Step 3（最重要）】

以下の仕様のDiscord Botを実装するための、
Claudeへの詳細な実装指示プロンプトを作成してほしい。

【確定した仕様】
（Step 2 で修正した最終要件をここに書く）

【技術スタック】
- Node.js 20.x
- discord.js v14
- dotenv（環境変数管理）
- fs（JSONファイルによるデータ永続化）
- GCP Compute Engine (e2-micro) でのPM2運用を想定

【Claudeへのプロンプトに必ず含める要素】
1. 完全なファイル構成（ディレクトリツリー）
2. 各ファイルの役割と実装すべき内容
3. データ設計（JSONの構造）
4. エラーハンドリングの方針
5. 日本語コメントを各ファイルに含めること
6. .env.example の内容
7. package.json の依存関係
8. 動作確認手順
9. 「全ファイルをZIPにまとめて提供すること」という指示

Claudeが一度の回答で完全に実装できる、
具体的で詳細なプロンプトを作成してください。
```

---

### 🤖 Step 4: Claude でZIPを生成する

Geminiが作ったプロンプトをそのまま Claude.ai に貼り付ける。

**送信前に必ず末尾に追加する一文:**

```
上記の仕様を完全に実装し、全ファイルをZIPにまとめてダウンロードできる形で提供してください。
実装の優先順位は「動くこと」「読みやすいこと」「拡張しやすいこと」の順です。
```

**Claudeが返してきたら:**

```
チェックポイント:
□ ファイル構成がStep 3の指示と一致しているか
□ package.json に必要なパッケージが入っているか
□ .env.example がある
□ README.md がある（あれば）
□ 各ファイルに日本語コメントがある
```

---

### ▶️ Step 5: 動かす → 詰まる → AIでデバッグ

```bash
# ZIPを解凍してから
cd your-bot-folder
cp .env.example .env

# .env を編集（DISCORD_TOKEN, CLIENT_ID, GUILD_ID を記入）
# テキストエディタで .env を開く

# 依存関係インストール
npm install

# スラッシュコマンドを登録
node src/deploy-commands.js

# Bot 起動
node src/index.js
```

**詰まったときのデバッグ依頼テンプレート（ChatGPT or Claude へ）:**

```
以下のエラーが発生しています。

【エラーメッセージ】（スタックトレース全文）
---
[ここにエラーをペースト]
---

【環境】
- Node.js: v20.x
- discord.js: v14.x  
- OS: [Windows / macOS / Ubuntu]

【やろうとしていたこと】
[1〜2行で説明]

【試したこと】
[すでに試した対処法]

原因と解決策を教えてください。
```

### 🔍 Step 6: Part 3 のコードと比較する

ここが最も学びになるステップだ。

AIが生成したコードと、Part 3 のコードを並べて比較する。

```
比較ポイント:
□ ファイル構成の違いはあるか？（どちらが読みやすいか）
□ クールダウン実装の方法が違うか？（どちらが良いか）
□ エラーハンドリングに差はあるか？
□ Embed のデザインが違うか？（どちらが見やすいか）
□ 自分が生成させたコードで「ここは直したい」と思う部分はどこか？
```

**AIに「コードレビュー」させる:**

```
【Claudeへのコードレビュー依頼】

添付のZIPは私がAIと協力して作ったDiscord Botのコードです。
以下の観点でコードレビューをしてください:

1. バグや潜在的な問題があるか
2. もっと良い書き方があるか（discord.js v14 の機能を活かせているか）
3. セキュリティ上の懸念はあるか
4. メモリリークの可能性はあるか（特に Map の管理）
5. GCPのe2-microで長期運用して問題が起きそうな箇所

改善点は具体的なコードで示してください。
```

## 4. よくある詰まりポイントとAI活用法

### 🔴 「Claudeが仕様と違うコードを生成した」

```
対処法: より具体的に再指示する

× 「もっと良くしてください」
○ 「src/commands/quiz.js の handleQuiz 関数を修正してください。
   現在は選択肢が毎回同じ順番で表示されます。
   正解の位置をランダムにしてください。
   修正箇所のコードのみ示してください。」
```

### 🔴 「ZIPを展開したら動かない」

```
デバッグの順序:
1. エラーメッセージを確認（node src/index.js の出力）
2. .env の値が正しいか確認
3. npm install を実行したか確認
4. deploy-commands.js を実行したか確認
5. それでも動かない → エラーをChatGPTに投げる
```

### 🔴 「Claudeの無料枠が切れた」

```
対処法（AI切り替え戦術）:

段階1: ChatGPT に同じZIPを投げてみる
  「このBotのコード（ZIP添付）を見て、○○の機能を追加してください」

段階2: Gemini に投げる
  Gemini.google.com でZIPを添付して同じ指示

段階3: Google AI Studio で試す
  Gemini API を直接叩いて、コードを生成させる
  → https://aistudio.google.com/

→ どのAIも「ZIP + 指示」の形式は理解できる。
  Claude の代替として使えるかを確認しておこう。
```

## 5. 自分で「難しいお題」に挑戦するための準備

### 🏗️ 本格的なBotを作るときの設計テンプレート

以下を ChatGPT か Gemini に貼り付けて、自分のお題に合わせて埋めてもらおう:

```
【設計書テンプレート】

# [Botの名前] 設計書

## 1. コンセプト
（一言でどんなBotか）

## 2. ターゲットユーザー
（誰が使うBot？どんな状況で使う？）

## 3. コアバリュー
（このBotがあることで何が解決されるか）

## 4. 機能一覧
| コマンド | 説明 | 優先度 |
|----------|------|--------|
| /xxx     | ...  | 必須   |
| /yyy     | ...  | あれば |

## 5. データ設計
（保存が必要なデータとその構造）

## 6. 技術スタック
- Node.js 20.x
- discord.js v14
- [その他必要なライブラリ]

## 7. インフラ
- GCP Compute Engine e2-micro
- PM2 でデーモン化

## 8. 制約・要件
- 無料枠内で動かす
- Bot の再起動でデータが失われない
- エラーが起きてもBotが落ちない
```

### 🎯 難易度を上げるときの「追加API」ガイド

```
【完全無料のAPI（APIキー不要または無料枠あり）】

天気データ:
  Open-Meteo API → https://open-meteo.com/
  → APIキー不要・完全無料・日本語対応

翻訳:
  Google Translate API → Google AI Studio 経由で無料枠あり
  DeepL API → 月50万文字まで無料

株価・為替:
  Alpha Vantage API → 無料枠あり（制限あり）

ランダム名言:
  ZenQuotes API → https://zenquotes.io/
  → APIキー不要・完全無料

ランダム画像:
  Unsplash API → 無料枠あり
  → 絵文字やEmbed画像として使える
```

## 6. 「AIゼロ生成」で身につくもの

このPartを一通り実践すると、以下のスキルが身についている:

```
✅ AIへの指示精度（1回の往復でほしいコードを得る能力）
✅ 要件定義の思考（「何を作るか」を言語化する能力）
✅ AIの使い分け（ChatGPT・Gemini・Claude それぞれの強みの体感）
✅ デバッグのプロセス（エラーの原因を仮説立てして潰す能力）
✅ コードレビューの視点（自分のコードの問題を見つける能力）
```

これらはすべて、**2026年のエンジニア市場で最も求められているスキル**だ。  
コードを書く速さではなく、「AIと協働できる設計力」が問われる時代が来ている。

## 7. 最終課題：あなただけのBotを作って公開する

### 🎯 最終課題の条件

```
以下を全て達成したら、本講座の卒業だ:

□ 自分で考えたお題のBotを1つ作る（おみくじ以外）
□ GCP Compute Engine（e2-micro）にデプロイして24時間稼働させる
□ 少なくとも1人の友人に使ってもらいフィードバックをもらう
□ そのフィードバックをもとに機能を1つ改善する
□ 改善は「AIへの指示」だけで行う（自分ではコードを書かない）
```

### 💬 「友人に見せる」ことの重要性

バイブコーディングエンジニアの最終的な成果物は「動くソフトウェア」だ。  
コードの品質ではなく、**誰かが使って喜ぶもの**を作ることがゴールだ。

友人の一言「あ、便利！」が、次の100時間の開発モチベーションになる。  
その体験を早く積むことが、長く続けるコツだ。
