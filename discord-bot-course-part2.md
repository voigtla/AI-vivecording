## 1. 今回作るBotの設計

### 🎯 お題：二段構成で学ぶ

本講座では **2種類のBot** を作る。

```
【Tier 1：おみくじBot（入門）】
  難易度: ★★☆☆☆
  目的: Discord.js の基本を学ぶ
  機能:
    - /omikuji コマンドで100パターンのおみくじを引く
    - 大吉〜大凶の7段階
    - 運勢ごとに色付きEmbedカード表示
    - 「今日は何回引いたか」のカウンター（メモリ内）
    - クールダウン機能（1時間に1回制限）

【Tier 2：天気×占いBot（中級）】
  難易度: ★★★★☆
  目的: 外部API連携・データ永続化・GCPデプロイを学ぶ
  機能:
    - /uranai [都市名] で天気と運勢を組み合わせた今日の運勢
    - Open-Meteo API（無料）で天気データ取得
    - 天気条件で運勢のウェイトを変える（晴れ→大吉出やすい等）
    - ユーザーごとの引いた回数をJSONで保存
    - GCP Cloud Run でホスティング
    - Uptime Kuma or Cloud Scheduler で常時稼働
```

> **なぜ2種類か？**  
> Tier 1 は「動く喜び」を最速で味わうため。  
> Tier 2 は「本物のBot」として誰かに見せられるレベルを目指すため。  
> どちらか片方だけ作っても構わない。

## 2. ChatGPTでアイデアを発散させる

### 💬 Step 1: 思想フェーズの実例

実際にChatGPTに投げたプロンプトの例:

```
【ChatGPTへの最初の投げかけ（例）】

Discord Botでおみくじを作りたい。
以下の要件がある:
- スラッシュコマンドで動く
- 大吉・中吉・小吉・吉・末吉・凶・大凶の7段階
- 各段階に複数のメッセージパターン（合計100個以上）
- 一人ひとりが楽しめる要素が欲しい

これに追加するとさらに面白くなるアイデアを10個以上出して。
技術的な実現可能性は無視していい。まずアイデアだけ。
```

**ChatGPTが返してくれるアイデア例:**

```
1. 「連続引き」機能 - 5回まとめて引いて一番良いのを表示
2. クールダウンタイマー表示 - 「次に引けるまであと2時間34分」
3. 引いた結果をサーバー内でランキング化
4. 季節・曜日で結果の確率を変える（月曜は大凶が出やすい等）
5. 「おみくじを結ぶ」ボタン - 悪い結果をキャンセルできる
6. 「引き直し券」- 週1回だけ大吉確定で引き直せる
7. 特定時間帯（深夜0時〜1時）は特別な「怪しいおみくじ」が出る
8. メンションして他のユーザーの運勢を「占う」機能
9. 占い結果をスクリーンショット風の画像で出力
10. 「おみくじガチャ履歴」- 自分の過去30日の引いた記録を見る
11. ギルド全体の今日の運勢バランスを表示（凶が多いと不吉な日）
12. 引いた瞬間のアニメーション演出（絵文字でアニメ）
```

```
   → node src/deploy-commands.js

7. Bot起動
   → node src/index.js

```

## 5. うまく動かないときのデバッグフロー

### 🔴 よくあるエラーと対処法

#### エラー1: `Error: TOKEN_INVALID`
```
原因: .env のトークンが間違っている
対処:
  1. Discord Developer Portal → Bot → Reset Token
  2. 新しいトークンを .env に貼り直す
  3. Bot token と Application ID を混同していないか確認
```

#### エラー2: スラッシュコマンドが表示されない
```
原因: deploy-commands.js を実行していない or GUILD_ID が違う
対処:
  1. node src/deploy-commands.js を実行
  2. Discordアプリを再起動（Ctrl+R）
  3. GUILD_ID が正しいか確認
     → サーバーを右クリック → 「IDをコピー」

注意: グローバルコマンドは反映に最大1時間かかる
      開発中はGUILD_IDを指定したギルドコマンドを使う
```

#### エラー3: `Missing Access` / `Missing Permissions`
```
原因: BotのパーミッションかIntentsの設定漏れ
対処:
  1. Discord Developer Portal → Bot → Privileged Gateway Intents
     → MESSAGE CONTENT INTENT をON
  2. OAuth2 → URL Generator で権限を再設定してBotを招待し直す
```

#### エラー4: Embedが表示されない
```
原因: Embedのフォーマットエラー or 色の指定が間違っている
対処:
  Discord.jsのEmbedカラーは16進数の数値で指定する
  × color: '#FF0000'
  ○ color: 0xFF0000
  ○ color: parseInt('FF0000', 16)
```

## 6. おみくじデータ設計（100パターン）

### 📊 運勢分布の設計

```javascript
// data/fortunes.js の設計思想

/*
 * 運勢の確率分布（合計100%）
 * 現実のおみくじを参考に、「吉」が最も多くなるよう設計
 */
const FORTUNE_RATES = {
  大吉: 10,   // 10%
  中吉: 15,   // 15%
  小吉: 15,   // 15%
  吉:   30,   // 30% ← 最多
  末吉: 15,   // 15%
  凶:   12,   // 12%
  大凶:  3,   //  3% ← 最少
};
// 合計: 100%
```

### 📝 各運勢のメッセージ設計

各運勢に複数のメッセージを持たせることで「毎回同じ」感をなくす:

```
大吉 (10パターン):
  - 「今日は何をやっても上手くいく日！積極的に動いて吉！」
  - 「絶好調！新しいことを始めるのに最適な日です。」
  - 「運気MAX！普段言えない一言を今日こそ伝えよう。」
  ...など10種類

凶 (15パターン):
  - 「今日は大人しくしていた方が無難かも...。」
  - 「少し慎重に。でも凶は結ぶことができるので安心して！」
  ...など15種類

大凶 (5パターン):
  - 「大凶！でも大丈夫。最悪の運気は上がるだけ！」
  - 「今日は家でゆっくりするのがベスト...かもしれない。」
  ...など5種類
```

> **合計100パターン達成の計算:**  
> 10+15+15+20+15+15+10 = 100パターン  
> （各運勢のパターン数は確率とは独立して設定可能）
